# АвтоМастер CRM

Веб-приложение для управления заказами, клиентской базой и записью на сервис для сети автосервисов.

## Описание проекта

Полнофункциональная CRM-система. Проект включает в себя три основных модуля:

1. **Дашборд** - интерактивная панель с графиками и статистикой
2. **Управление заказами** - виртуализированный список с фильтрацией и поиском
3. **Календарь записи** - drag-and-drop планирование визитов клиентов

## Технологический стек

### Основные технологии
- **React 18** - библиотека для построения UI
- **TypeScript** - типизация и безопасность кода
- **Redux Toolkit** - централизованное управление состоянием
- **React Router v6** - маршрутизация
- **Tailwind CSS** - стилизация
- **Vite** - сборщик проекта

### Библиотеки
- **React Hook Form + Yup** - валидация форм
- **React Window** - виртуализация больших списков (1500+ записей)
- **React Beautiful DnD** - drag-and-drop функциональность
- **Recharts** - графики и диаграммы
- **date-fns** - работа с датами
- **Axios** - HTTP-клиент (готов к интеграции с API)

## Ключевые особенности

### 1. Оптимизация производительности
- **Виртуализация списка заказов** через react-window для мгновенного рендеринга 1500+ записей
- **Мемоизация компонентов** для предотвращения лишних перерисовок
- Оптимизированные вычисления через useMemo и useCallback

### 2. State Management
- Redux Toolkit для централизованного хранилища
- Разделение логики на отдельные slices (orders, calendar)
- Типизированные hooks (useAppDispatch, useAppSelector)

### 3. Формы и валидация
- React Hook Form для высокопроизводительных форм
- Yup схемы для валидации на стороне клиента
- Пользовательские сообщения об ошибках

### 4. UX/UI
- Современный минималистичный дизайн
- Адаптивная верстка (desktop-first с поддержкой планшетов)
- Интерактивные элементы с визуальной обратной связью

### 5. Drag-and-Drop календарь
- Перетаскивание записей между днями и временными слотами
- Визуальная индикация при перетаскивании
- Автоматическое обновление времени встречи

## Решенные технические проблемы

### Проблема: Медленный рендеринг списка заказов
**Решение:**
- Внедрена виртуализация через react-window
- Рендерится только видимая часть списка (примерно 10-15 элементов)
- Время рендера сократилось с 3-4 секунд до мгновенного

### Проблема: Лишние перерисовки компонентов
**Решение:**
- Мемоизация компонента OrderRow через React.memo
- Использование useMemo для вычисляемых значений
- Оптимизированные селекторы Redux

### Проблема: Сложная логика фильтрации и сортировки
**Решение:**
- Вынесена логика в Redux slice
- Централизованная функция applyFilters
- Автоматическое применение фильтров при изменении параметров
